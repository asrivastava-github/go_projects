# awsct Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Go CLI + MCP server tool (`awsct`) that queries AWS CloudTrail events by action, user, or resource.

**Architecture:** Single Go binary with cobra subcommands for CLI mode and an MCP stdio server mode. Core CloudTrail query logic lives in `pkg/cloudtrail/`, shared by both CLI and MCP layers. Auth follows the `ip-finder` pattern (SDK v2 default chain + aws-vault re-exec for CLI).

**Tech Stack:** Go 1.23+, AWS SDK v2 (cloudtrail, sts, config), cobra, text/tabwriter, mark3labs/mcp-go

**Spec:** `docs/superpowers/specs/2026-04-17-awsct-design.md`

### Task 2: Duration Parser

**Files:**
- Create: `awsct/pkg/cloudtrail/duration.go`

- [ ] **Step 1: Create duration.go**

Parse human-friendly duration strings into `time.Duration`:
- `30m` → 30 minutes
- `1h` → 1 hour
- `24h` → 24 hours
- `7d` → 7 days
- `90d` → 90 days (max)

```go
package cloudtrail

import (
	"fmt"
	"strconv"
	"strings"
	"time"
)

const MaxLookback = 90 * 24 * time.Hour // CloudTrail API limit

func ParseDuration(s string) (time.Duration, error) {
	s = strings.TrimSpace(s)
	if s == "" {
		return 24 * time.Hour, nil // default
	}

	// Handle "d" suffix (days) — not supported by time.ParseDuration
	if strings.HasSuffix(s, "d") {
		days, err := strconv.Atoi(s[:len(s)-1])
		if err != nil {
			return 0, fmt.Errorf("invalid duration: %s", s)
		}
		d := time.Duration(days) * 24 * time.Hour
		if d > MaxLookback {
			return MaxLookback, nil
		}
		return d, nil
	}

	d, err := time.ParseDuration(s)
	if err != nil {
		return 0, fmt.Errorf("invalid duration: %s", s)
	}
	if d > MaxLookback {
		return MaxLookback, nil
	}
	return d, nil
}
```

```go
package cloudtrail

import (
	"context"
	"fmt"

	"github.com/aws/aws-sdk-go-v2/config"
	ct "github.com/aws/aws-sdk-go-v2/service/cloudtrail"
)

type Client struct {
	ct     *ct.Client
	region string
}

func NewClient(ctx context.Context, region, profile string) (*Client, error) {
	opts := []func(*config.LoadOptions) error{
		config.WithRegion(region),
	}
	if profile != "" {
		opts = append(opts, config.WithSharedConfigProfile(profile))
	}

	cfg, err := config.LoadDefaultConfig(ctx, opts...)
	if err != nil {
		return nil, fmt.Errorf("failed to load AWS config: %w", err)
	}

	return &Client{
		ct:     ct.NewFromConfig(cfg),
		region: region,
	}, nil
}
```

### Task 4: Core Lookup Logic

**Files:**
- Create: `awsct/pkg/cloudtrail/lookup.go`

- [ ] **Step 1: Create lookup.go with three lookup functions**

Implement `LookupByAction`, `LookupByUser`, `LookupByResource`. All three follow the same pattern — they only differ in the `LookupAttribute` type. Use a shared internal `lookup` function that handles:

1. Build `LookupEventsInput` with the attribute type and value
2. Set `StartTime` = `now - duration`, `EndTime` = `now`
3. Set `MaxResults` = min(50, remaining limit) per page
4. Paginate using `NextToken` until no more pages or limit reached
5. Parse each event into the `Event` struct
6. Handle throttling with exponential backoff (check for `ThrottlingException`)

Key implementation details:
- CloudTrail `LookupEvents` returns `[]types.Event` where each event has `CloudTrailEvent` (JSON string with full details)
- Parse `CloudTrailEvent` JSON to extract `sourceIPAddress`, `readOnly`, `userIdentity`
- The `Resources` field is directly available on the event as `event.Resources`
- Use `types.LookupAttributeKey*` constants: `LookupAttributeKeyEventName`, `LookupAttributeKeyUsername`, `LookupAttributeKeyResourceName`

```go
package cloudtrail

import (
	"context"
	"encoding/json"
	"fmt"
	"math"
	"time"

	"github.com/aws/aws-sdk-go-v2/aws"
	ct "github.com/aws/aws-sdk-go-v2/service/cloudtrail"
	"github.com/aws/aws-sdk-go-v2/service/cloudtrail/types"
)

func (c *Client) LookupByAction(ctx context.Context, action string, params LookupParams) ([]Event, error) {
	return c.lookup(ctx, types.LookupAttributeKeyEventName, action, params)
}

func (c *Client) LookupByUser(ctx context.Context, username string, params LookupParams) ([]Event, error) {
	return c.lookup(ctx, types.LookupAttributeKeyUsername, username, params)
}

func (c *Client) LookupByResource(ctx context.Context, resource string, params LookupParams) ([]Event, error) {
	return c.lookup(ctx, types.LookupAttributeKeyResourceName, resource, params)
}
```

The internal `lookup` function:
- Builds the `LookupEventsInput` with one `LookupAttribute`
- Loops with pagination (`NextToken`)
- For each `types.Event`, parses the `CloudTrailEvent` JSON string to extract `sourceIPAddress` and `readOnly`
- Builds `[]Event` from the results
- On throttle errors, waits with exponential backoff (base 500ms, max 8s, max 5 retries)
- Stops when limit is reached or no more pages

The `CloudTrailEvent` JSON string has this structure (parse only what we need):
```go
type cloudTrailEventDetail struct {
	SourceIPAddress string `json:"sourceIPAddress"`
	ReadOnly        bool   `json:"readOnly"`
}
```

Create the `serve-mcp` subcommand that starts a stdio MCP server with three tools:

```go
var serveCmd = &cobra.Command{
	Use:   "serve-mcp",
	Short: "Start MCP server (stdio transport)",
	RunE:  runServe,
}
```

`runServe` function:
1. Create `server.NewMCPServer("awsct", "1.0.0", server.WithToolCapabilities(false))`
2. Register three tools:

**Tool: lookup_by_action**
- Parameters: `action` (string, required), `last` (string, optional, default "24h"), `region` (string, optional, default "us-east-1")
- Handler: creates CloudTrail client, calls `LookupByAction`, returns JSON results as `mcp.NewToolResultText`

**Tool: lookup_by_user**
- Parameters: `username` (string, required), `last` (string, optional), `region` (string, optional)
- Handler: calls `LookupByUser`

**Tool: lookup_by_resource**
- Parameters: `resource` (string, required), `last` (string, optional), `region` (string, optional)
- Handler: calls `LookupByResource`

3. Start with `server.ServeStdio(s)`

Each handler:
1. Extract parameters using `request.RequireString()` / `mcp.ParseString()`
2. Parse duration with `cloudtrail.ParseDuration`
3. Create client with empty profile (MCP mode uses ambient creds)
4. Execute lookup
5. Marshal results to JSON
6. Return `mcp.NewToolResultText(jsonString)`

**Important:** Do NOT include aws-vault re-exec logic in MCP mode. MCP uses ambient credentials only.

Reference: The mcp-go API from the search results:
```go
tool := mcp.NewTool("lookup_by_action",
    mcp.WithDescription("Find who performed a specific AWS API action"),
    mcp.WithString("action", mcp.Required(), mcp.Description("AWS API action name, e.g. DeleteBucket")),
    mcp.WithString("last", mcp.Description("Time window, e.g. 1h, 24h, 7d. Default: 24h")),
    mcp.WithString("region", mcp.Description("AWS region. Default: us-east-1")),
```
